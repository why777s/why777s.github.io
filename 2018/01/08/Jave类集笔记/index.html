  2. 常用的扩展方法

二. Set接口

set接口也是Collection接口的子接口，但是与Collection或List接口不同的是，Set接口中不能加入重复的元素。

Set接口常用子类

  1. 散列的存放：HashSet
    特点：不存放重复元素；采用散列存储的方式，故没有顺序
  2. 有序的存放：TreeSet
    对输入的数据进行有序排列（TreeSet实现了SortedSet接口）；若要自定义排序顺序，只需覆写compareTo()方法，自行指定排序规则。

    SortedKey接口主要用于排序操作，即实现此接口的子类都属于排序的子类。

三. 集合的输出

1.迭代输出:Iterator

Iterator是专门的迭代输出接口，所谓的迭代输出就是将元素一个个进行判断，判断其是否有内容，若有则把内容取出。使用Iterator接口时需指定泛型。此接口定义了如下三个方法：

方法 描述
public boolean hasNext() 判断是否有下一个值
public E next() 取出当前元素
public void remove() 移出当前元素

除了输出内容，还可用于删除当前的内容，直接使用remove()即可。

2.双向迭代输出:ListIterator

Iterator仅是由前向后单向输出，而此时如果想实现由后向前或是由前向后的双向输出，则必须使用Iterator的子接口——ListIterator。
若想从后向前遍历，则通过public boolean hasPrevious()判断是否有上一条件以及通过public E previous()取出当前元素。

3. foreach

foreach除了可以完成数组的输出，对于集合也同样支持.

四. Map接口

map接口每次操作的是一对对象，即二元偶对象，Map接口中的每个元素都使用“Key-value”的形式存储在集合中。在Map接口上也应用了泛型，必须同时设置好key和value的类型。

方法 描述
public void clear() 清空
public boolean containsKey 判断指定的key是否存在
public boolean containsValue 判断指定的Value是否存在
public Set> entrySet() 将Map对象变为set集合
public boolean equals(Object o) 对象比较
public V get(Object key) 根据key取得value
public V put(K key,V value) 向集合中增加元素
public V remove(Object key) 根据key删除value
public int size() 求集合的长度

1. .Entry接口

Map.entry接口是Map内部定义的一个接口，专门用来保存key-value内容。在Map接口的操作中，所有的内容都是通过key-value的形式保存数据的，那么对于集合来说，实际上是将”key-value”的数据保存在了Map.Entry接口的实例之后，再在Map集合中插入的是一个Map.Entry的实例化对象。

2. Map接口的常用子类

  1. HashTable：无序存放，key不允许重复
  2. HashMap：无序存放，key不允许重复

    HashMap和HashTable的区别：
    HashMap于JDK1.2之后推出，属于新的操作类；相较于采用同步处理方式的HashTable，HashMap采用异步处理方式，性能更高；HashMap允许将key设为null。

  3. TreeMap：可以排序的Map集合，按集合中的key排序，key不允许重复

    TreeMap是实现了SortedMap的子类，只要是实现了SortedMap接口的子类，都属于排序的子类。
